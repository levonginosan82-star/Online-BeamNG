# 🚗 Online BeamNG.drive — Мультиплеерный мод

> **Клиент-серверный мод для игры BeamNG.drive, добавляющий полноценный онлайн-мультиплеер.**
> Сервер на Node.js/TypeScript, клиент — Lua-мод внутри игры, управление через веб-интерфейс.

## 🚀 Быстрый старт

### 🐱 Самый быстрый способ (Windows)

Просто запустить **`launcher\start_all.bat`** — он автоматически:
1. Проверит/установит зависимости (`npm install`)
2. Соберёт TypeScript (`npm run build`)
3. Запустит сервер
4. Откроет BeamNG.drive (если установлен через Steam)

```bash
# Два клика мышкой — и всё работает
.\launcher\start_all.bat
```

После запуска открыть в браузере: **http://localhost:30815**

## 🖥 Установка сервера

### 📋 Требования

| Компонент | Версия |
|-----------|--------|
| [Node.js](https://nodejs.org) | ≥ 18.0.0 |
| npm | ≥ 9.0.0 |
| BeamNG.drive | 0.38.x |
| ОС | Windows (основная), Linux |

### 🔧 Пошаговая установка

```bash
# 1. Перейти в папку сервера
cd server

# 2. Установить зависимости
npm install

# 3. Собрать TypeScript
npm run build

# 4. Запустить сервер
npm start
```

### ⚙️ Настройка через веб-интерфейс

1. Открыть **http://localhost:30815**
2. Перейти на вкладку **Configuration**
3. Настроить параметры:
   - **Server Name** — название сервера
   - **Port** — порт для игроков (по умолч. 30814)
   - **Max Players** — макс. количество игроков
   - **Password** — пароль (пусто — без пароля)
   - **Map** — карта по умолчанию
   - **Damage Sync** — синхронизация повреждений
4. Нажать **Save Configuration**
5. Перейти на вкладку **Dashboard**
6. Нажать **Start**

## 🎮 Установка клиента (мода)

### 📥 Автоматическая установка

```powershell
# PowerShell (права администратора НЕ нужны)
.\scripts\install_mod.ps1
```

Или:
```cmd
:: CMD
.\scripts\install_mod.bat
```

Скрипт сам:
1. Найдёт установленную BeamNG.drive
2. Скопирует мод в `%USERPROFILE%\Documents\BeamNG.drive\mods\`
3. Создаёт ярлык на рабочем столе

### 📁 Ручная установка

1. Скопировать папку `client\mods\OnlineBeamNG` в:
   ```
   %USERPROFILE%\Documents\BeamNG.drive\mods\
   ```
   > 💡 Быстрый доступ: нажать `Win+R`, вставить `%USERPROFILE%\Documents\BeamNG.drive\mods`

2. Запустить BeamNG.drive
3. В главном меню появится кнопка **"Online BeamNG"**
4. Ввести IP сервера и нажать **Connect**

### ✅ Проверка установки

После установки мода в игре должен появиться пункт меню "Online BeamNG". Если его нет:
- Проверьте версию BeamNG.drive (нужна 0.38.x)
- Убедитесь, что мод включён в менеджере модов игры
- Проверьте логи игры (`%USERPROFILE%\Documents\BeamNG.drive\beamng.log`)

## 🌐 Веб-админка

> **URL:** http://localhost:30815

Веб-админка — это полноценный интерфейс для управления сервером без необходимости редактировать файлы вручную.

### 📊 Dashboard (Главная)

| Элемент | Описание |
|---------|----------|
| Статус сервера | Running / Stopped с цветным индикатором |
| Статистика | Количество игроков, машин, аптайм |
| Информация | Карта, имя сервера, порт админки |
| Кнопки | **Start / Stop / Restart** |

### ⚙️ Configuration (Настройки)

| Параметр | Тип | Описание |
|----------|-----|----------|
| Server Name | текст | Название сервера (до 64 символов) |
| Description | текст | Описание сервера |
| Port | число | Порт для подключения игроков (1-65535) |
| Host | текст | IP для прослушивания (0.0.0.0 — все интерфейсы) |
| Max Players | число | Максимум игроков (1-64) |
| Password | текст | Пароль (пусто — без пароля) |
| Map | текст | Название карты по умолчанию |
| Max Vehicles | число | Макс. машин на игрока |
| Tick Rate | число | Тактов в секунду (10-60) |
| Log Level | выбор | Уровень логов: Debug/Info/Warn/Error |
| Damage Sync | вкл/выкл | Синхронизация повреждений |
| Node Sync | вкл/выкл | Синхронизация нод (эксперимент.) |
| Mod Check | вкл/выкл | Проверка модов у игроков |
| Private | вкл/выкл | Приватный сервер |

### 👥 Players (Игроки)

Таблица с информацией:
- **ID** — уникальный идентификатор
- **Name** — ник игрока
- **Role** — роль (admin/user)
- **Ping** — задержка в мс

### 📜 Logs (Логи)

- Логи в реальном времени
- Авто-скролл (вкл/выкл)
- Кнопка очистки
- Цветная подсветка уровней (info/warn/error)

## 🔌 REST API

Все эндпоинты: **`http://localhost:30815/api/`**

### Эндпоинты

| Метод | Endpoint | Описание | Пример ответа |
|-------|----------|----------|--------------|
| `GET` | `/status` | Статус сервера | `{ running: true, players: 3, ... }` |
| `GET` | `/config` | Текущая конфигурация | `{ port: 30814, name: "...", ... }` |
| `PUT` | `/config` | Обновить конфигурацию | Тело: `{ name: "New Name" }` |
| `POST` | `/start` | Запустить game-сервер | `{ success: true }` |
| `POST` | `/stop` | Остановить game-сервер | `{ success: true }` |
| `POST` | `/restart` | Перезапустить game-сервер | `{ success: true }` |
| `GET` | `/logs?count=50` | Последние N логов | `{ logs: [...] }` |
| `GET` | `/players` | Список игроков | `{ players: [...], count: 3 }` |
| `GET` | `/config-template` | Шаблон конфигурации | `{ port: 30814, ... }` |

### Примеры использования

```bash
# Получить статус
curl http://localhost:30815/api/status

# Запустить сервер
curl -X POST http://localhost:30815/api/start

# Обновить название сервера
curl -X PUT http://localhost:30815/api/config \
  -H "Content-Type: application/json" \
  -d '{"name":"My Server"}'

# Получить последние 10 логов
curl "http://localhost:30815/api/logs?count=10"
```

## 📡 Протокол WebSocket

Формат всех сообщений — **JSON**:

```json
{
  "type": "MessageType",
  "data": { ... },
  "sender": "player-uuid",
  "timestamp": 1716543210000
}
```

### Типы сообщений

| Тип | Направление | Описание | Данные |
|-----|------------|----------|--------|
| `Hello` | C→S | Приветствие | `{ clientVersion, protocolVersion }` |
| `HelloResponse` | S→C | Версия сервера | `{ serverVersion, protocolVersion, serverName }` |
| `Auth` | C→S | Авторизация | `{ username, password }` |
| `AuthResponse` | S→C | Результат | `{ success, playerId, role }` |
| `PlayerJoin` | S→C | Игрок зашёл | `{ id, name, role, ping }` |
| `PlayerLeave` | S→C | Игрок вышел | `{ playerId, name }` |
| `PlayerList` | S→C | Список игроков | `{ players: [...] }` |
| `VehicleUpdate` | C↔S | Состояние машины | `{ id, position, rotation, velocity, ... }` |
| `VehicleSpawn` | C→S | Спавн машины | `{ jBeam, position, rotation, ... }` |
| `VehicleDespawn` | C→S | Удаление машины | `{ vehicleId }` |
| `ChatMessage` | C↔S | Сообщение в чат | `{ sender, senderId, message, type }` |
| `VehicleDamage` | C↔S | Повреждения | `{ vehicleId, nodes, damage }` |
| `VehicleInput` | C↔S | Ввод игрока | `{ vehicleId, steering, throttle, brake }` |
| `VehicleElectrics` | C↔S | Электрика | `{ vehicleId, lights, ... }` |
| `VehicleNodes` | C↔S | Позиции нод | `{ vehicleId, nodes: [...] }` |
| `Kick` | S→C | Кик | `{ reason }` |
